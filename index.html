Wi-Fi Protocols

Wi-Fi networks operate on IEEE 802.11 standards, which have evolved over time:

  • 802.11a/b/g - Early standards with limited security
  • 802.11n - Improved speeds and some security enhancements
  • 802.11ac/ax - Modern standards with better security features

Security Types

Wi-Fi networks use various security protocols:

  • WEP - Outdated and highly vulnerable
  • WPA - Improved but still has vulnerabilities
  • WPA2 - Current standard with AES encryption
  • WPA3 - Latest standard with enhanced security

Common Vulnerabilities

Wi-Fi networks can be vulnerable to various issues:

  • Weak encryption (particularly in WEP and early WPA)
  • Default or weak passwords
  • Unpatched firmware vulnerabilities
  • Deauthentication attacks (which we'll study safely)

Module 3: Understanding Network Monitoring

+

Network monitoring is a crucial skill for security professionals. Let's learn how to properly monitor network activity in a controlled environment.

Setting Up a Network Interface in Monitor Mode

The first step in network analysis is setting up your wireless interface to monitor network traffic:

sudo airmon-ng check kill
Killing these processes: PID Name 1234 NetworkManager
sudo airmon-ng start wlan0
PHY Interface Driver Chipset phy0 wlan0mon ath9k Qualcomm Atheros

Explanation

These commands:

  • Kill processes that might interfere with monitor mode
  • Enable monitor mode on the wireless interface (wlan0)
  • The interface is renamed to wlan0mon when in monitor mode

Module 4: Capturing Network Traffic

+

Once your interface is in monitor mode, you can capture and analyze network traffic to identify vulnerabilities.

Using airodump-ng for Traffic Capture

Airodump-ng is a tool that captures packets and displays information about nearby networks:

sudo airodump-ng wlan0mon
BSSID PWR Beacons #Data CH MB ENC CIPHER AUTH ESSID 00:11:22:33:44:55 -70 542 124 6 54e WPA2 CCMP PSK TestNetwork AA:BB:CC:DD:EE:FF -85 231 42 1 54e WPA TKIP PSK OtherNetwork

Explanation

This command shows all wireless networks in range, including:

  • BSSID - The MAC address of the access point
  • PWR - Signal strength
  • CH - The channel the network is operating on
  • ENC - The encryption type (WEP, WPA, WPA2)
  • ESSID - The network name

Focusing on a Specific Network

To analyze a specific network, you can filter airodump-ng:

sudo airodump-ng -c 6 --bssid 00:11:22:33:44:55 -w capture wlan0mon
BSSID PWR Beacons #Data CH MB ENC CIPHER AUTH ESSID 00:11:22:33:44:55 -70 542 124 6 54e WPA2 CCMP PSK TestNetwork STATION PWR Rate Lost Frames BSSID PROBES 11:22:33:44:55:66 -75 0 - 1 0 42 00:11:22:33:44:55

Explanation

This command:

  • Filters to show only channel 6
  • Focuses on the specific BSSID (access point)
  • Saves the capture to files with the prefix "capture"
  • Also shows connected clients (STATION)

Understanding Deauthentication Frames

Deauthentication frames are management frames in Wi-Fi that normally help clients disconnect properly. However, they can be used to analyze network behavior:

Legitimate Uses for Security Professionals:

  • Testing network resilience against disconnections
  • Analyzing how devices reconnect to networks
  • Troubleshooting connection issues
  • Testing network failover mechanisms

Simulation Environment

Let's simulate analyzing a network using aireplay-ng in our controlled environment:

sudo aireplay-ng -0 1 -a 00:11:22:33:44:55 -c 11:22:33:44:55:66 wlan0mon
15:22:48 Waiting for beacon frame (BSSID: 00:11:22:33:44:55) on channel 6 15:22:49 Sending deauthentication frame to client (11:22:33:44:55:66) 15:22:49 Sent 1 deauthentication frame

Explanation of Parameters

  • -0: Specifies a deauthentication packet type
  • 1: Number of packets to send (minimal for testing)
  • -a: Target access point's MAC address
  • -c: Target client's MAC address
  • wlan0mon: Interface to use

Module 6: Ethical Considerations and Best Practices

+

As security professionals, ethics must guide our work. This module covers the ethical framework for network security testing.

Ethical Guidelines for Network Security Testing

  • Permission: Always obtain explicit written permission before testing any network
  • Scope: Clearly define and respect the boundaries of your testing
  • Documentation: Maintain detailed logs of all testing activities
  • Minimal Impact: Conduct tests in ways that minimize disruption to legitimate users
  • Confidentiality: Protect any sensitive information discovered during testing
  • Responsible Disclosure: Report vulnerabilities to appropriate parties

Legal Considerations

Network security testing without permission can violate various laws, including:

  • Computer Fraud and Abuse Act (US)
  • Computer Misuse Act (UK)
  • Similar cybercrime laws in most countries

Penalties can include fines and imprisonment.

Securing Your Own Network

Based on what we've learned, here are ways to better secure your networks:

Wi-Fi Security Best Practices

  • Use WPA3 when available, or WPA2 with a strong password
  • Enable Protected Management Frames (PMF) if supported
  • Change default router credentials
  • Keep router firmware updated
  • Consider MAC address filtering (though not foolproof)
  • Use a guest network for visitors
  • Consider implementing 802.1X authentication for enterprise networks
